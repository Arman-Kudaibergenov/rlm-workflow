# Варианты LLM-бэкенда

RLM-Toolkit использует LLM для двух задач:
- **Эмбеддинги** — векторное представление фактов для семантического поиска
- **Суммаризация** (опционально) — для routing-логики

Мощный GPU **не нужен**. 95% возможностей доступно на любом железе.

---

## Вариант 1: FastEmbed (встроенный, нулевая настройка)

Рекомендуется для начала. Работает на CPU, не требует API-ключей.

```env
RLM_EMBEDDING_PROVIDER=fastembed
# Больше ничего не нужно
```

В оригинальном RLM-Toolkit автор использует `all-MiniLM-L6-v2` (~80 МБ) — быстрая и лёгкая, но **только английский**.

Мы у себя в инфраструктуре используем `qwen3-embedding:8b` через Ollama на отдельном GPU-сервере (см. раздел "Наша инфраструктура" ниже).

В Docker-образе по умолчанию стоит `paraphrase-multilingual-MiniLM-L12-v2` (~471 МБ) — мультиязычная модель (50+ языков, включая русский), работает на CPU без GPU. Вы всегда можете заменить её на любую удобную через переменную `RLM_EMBEDDING_MODEL` в `.env`.

**Используйте если:** хотите нулевую настройку, нет GPU, не хотите платить за API.

---

## Вариант 2: OpenAI API (облако)

Самый простой вариант если уже есть аккаунт OpenAI.
`text-embedding-3-small` стоит ~$0.02 за 1M токенов — копейки для личного использования.

```env
RLM_EMBEDDING_PROVIDER=openai
OPENAI_API_KEY=sk-...
RLM_EMBEDDING_MODEL=text-embedding-3-small
```

**Используйте если:** уже платите за OpenAI, хотите облачное удобство без локального железа.

---

## Вариант 3: Ollama (локально, CPU или GPU)

[Ollama](https://ollama.com) запускает модели локально. Работает на CPU, быстрее с GPU.

```bash
# Установить Ollama
curl -fsSL https://ollama.com/install.sh | sh   # Linux/Mac
# Windows: скачать installer с ollama.com

# Скачать модель эмбеддингов (работает на CPU, ~274 МБ)
ollama pull nomic-embed-text
```

```env
RLM_EMBEDDING_PROVIDER=ollama
OLLAMA_BASE_URL=http://localhost:11434
RLM_EMBEDDING_MODEL=nomic-embed-text
```

**Используйте если:** хотите полностью локальный стек без облачных зависимостей.

## Какой LLM использовать для самого Claude?

RLM нужны только эмбеддинги. Но если хотите запускать вспомогательные модели локально:

| Железо | Модель | RAM/VRAM | Качество |
|--------|--------|----------|----------|
| Только CPU (4 ГБ RAM) | `qwen2.5:0.5b` | ~1 ГБ | Минимальное |
| CPU / встройка (8 ГБ) | `phi3.5-mini` | ~2.5 ГБ | Хорошее |
| GPU 4 ГБ (GTX 1650+) | `llama3.2:3b` | ~2 ГБ | Хорошее |
| GPU 8 ГБ (RTX 3060+ / RX 6600+) | `qwen3:8b` | ~5 ГБ | Отличное |
| Нет GPU → облако | `gpt-4o-mini` | — | Отличное |

Для большинства пользователей оптимально: **FastEmbed + OpenAI API** (или Anthropic API):
- FastEmbed обрабатывает поиск локально (бесплатно, приватно)
- API используется только для самого Claude

## Наша инфраструктура (для справки)

Описание для тех, кто хочет понять нашу конкретную настройку:

- **RLM-сервер:** Proxmox LXC (2 vCPU, 4 ГБ RAM)
- **Ollama-сервер:** отдельный LXC с GPU passthrough AMD RX 7700 XT через Vulkan
- **Модели:** `qwen3:8b` для inference, `qwen3-embedding:8b` для эмбеддингов
- **Флаг:** `OLLAMA_VULKAN=true`

Это избыточно для RLM в одиночку. FastEmbed покрывает 95% использования.
