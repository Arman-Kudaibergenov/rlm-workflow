## Детальное сравнение

### EncrEor/rlm-claude

Наиболее близкий аналог по концепции. Тоже решает проблему контекста для Claude Code через MCP + pre-compact hook.

**Преимущества их решения:**
- Проще в установке (`pip install mcp-rlm-server[all]`)
- Меньше конфигурации
- Поддержка французского и английского языков из коробки

**Чего не хватает:**
- Нет иерархических фактов (L0–L3)
- Нет каузальных решений
- Нет PENDING-паттерна с project-scoping
- Нет мульти-агентного workflow
- Нет мониторинга контекста (statusline + context-monitor)

### Mem0

Популярная облачная память для агентов.

**Преимущества:**
- Работает с любым LLM (не только Claude)
- Развитая SDK для разных языков
- Автоматическое извлечение фактов из разговора (не нужно явно сохранять)

**Недостатки:**
- Основной вариант — платный облачный сервис
- Self-hosted сложнее в настройке
- Нет workflow-ритуалов специфичных для разработки
- Нет интеграции с Claude Code hooks

### Zep

Долгосрочная память для агентов с temporal reasoning.

**Преимущества:**
- Отслеживает изменения фактов во времени
- Работает с любым LLM
- Хорошая документация

**Недостатки:**
- Нет специфики для coding workflow
- Требует настройки под каждый use case
- Нет готовых ритуалов для разработки

### MemGPT / OpenMemGPT (Letta)

Полноценный фреймворк для агентов с управлением памятью.

**Преимущества:**
- Самая продвинутая архитектура памяти
- Автономное управление контекстом

**Недостатки:**
- Высокий порог входа
- Предназначен для создания агентов, а не для работы с ними
- Не интегрируется с Claude Code напрямую

### Только CLAUDE.md

Самое простое решение — файл с инструкциями в корне проекта.

**Подходит для:**
- Небольших проектов
- Простых convention-based правил
- Когда не нужна история решений

**Не подходит когда:**
- Нужно больше 200 строк контекста
- Несколько проектов с разным контекстом
- Нужна история "почему так сделано"
- Задачи длятся несколько дней

## Вывод: когда использовать этот workflow

✅ **Подходит если:**
- Работаете с Claude Code регулярно (ежедневно)
- Задачи занимают несколько сессий / дней
- Важно сохранять историю архитектурных решений
- Используете мульти-агентный подход
- Хотите полностью self-hosted решение

❌ **Не подходит если:**
- Нужна поддержка других LLM кроме Claude
- Ищете облачный сервис без своего сервера
- Задачи простые и укладываются в одну сессию
