# Безопасность и приватность

> RLM хранит всё что вы обсуждаете с Claude. Прочитайте этот раздел перед деплоем.

## Что хранится в RLM

Всё что вы передаёте в `rlm_add_hierarchical_fact` и `rlm_record_causal_decision` — хранится персистентно.

В рабочем процессе это включает:
- Описания задач и их статус
- Принятые архитектурные решения
- Технические детали: IP-адреса, порты, имена серверов
- Названия файлов и фрагменты кода
- Ошибки и способы их решения
- Контекст проектов

**Что туда НЕ попадает автоматически:**
- Полное содержимое файлов (только пути)
- Пароли и токены (если вы их не вставляли в чат)
- Весь текст разговора (только то что явно сохраняется через RLM-инструменты)

## Основные риски

### 1. Секреты в контексте разговора

Если вы вставили в чат `OPENAI_API_KEY=sk-abc123` или пароль от БД — Claude может сохранить это в RLM факте "технические детали".

**Правило:** никогда не вставляйте секреты в чат. Используйте переменные окружения или ссылайтесь на файлы (`.env`, `credentials.md`).

### 2. Открытый порт 8200

По умолчанию RLM-сервер не требует аутентификации. Если порт доступен из интернета — любой может читать и писать вашу память.

```bash
# НЕПРАВИЛЬНО: доступен из интернета
docker run -p 8200:8200 ...

# ПРАВИЛЬНО: только localhost
docker run -p 127.0.0.1:8200:8200 ...
```

Для удалённого сервера используйте:
- Firewall (разрешить только доверенные IP)
- SSH-туннель: `ssh -L 8200:localhost:8200 user@server`
- VPN

### 3. Данные на диске

Docker volume `rlm-data` содержит все факты в незашифрованном виде.

Рекомендации:
- Для продакшна: включите шифрование диска на уровне ОС
- Регулярно делайте бэкапы (`docker cp rlm:/data ./backup`)
- Настройте TTL для чувствительных фактов (`ttl_days=7`)

## Настройки для командной работы

Если несколько человек используют один RLM-сервер:

1. **Изоляция проектов** — используйте `[project: name]` теги в фактах
2. **Один сервер на команду** — не один на компанию
3. **Аудит доступа** — RLM пока не имеет встроенной авторизации по пользователям

## TTL для временных фактов

Чувствительные данные можно хранить с ограниченным временем жизни:

```python
rlm_add_hierarchical_fact(
    content="Токен доступа к тестовому стенду: ...",
    domain="workflow",
    level=1,
    ttl_days=1   # исчезнет через день
)
```

PENDING-факты рекомендуем хранить 30 дней (`ttl_days=30`).

## Рекомендуемая конфигурация для продакшна

```yaml
# docker-compose.yml
services:
  rlm:
    ports:
      - "127.0.0.1:8200:8200"   # только localhost
    environment:
      - RLM_MAX_FACT_TTL_DAYS=90  # автоочистка старых фактов
```

И закройте порт на файрволе:
```bash
ufw deny 8200      # Linux UFW
ufw allow from 192.168.1.0/24 to any port 8200  # разрешить только LAN
```
