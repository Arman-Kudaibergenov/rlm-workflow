# Установка

## Требования

- Docker и Docker Compose (рекомендуется)
- Или Python 3.10+ (без Docker)
- Claude Code CLI

Минимум ресурсов для RLM-сервера: **2 CPU, 2 ГБ RAM**.

---

## Вариант 1: Docker (рекомендуется)

Самый простой способ — одна команда:

```bash
docker run -d \
  --name rlm \
  --restart unless-stopped \
  -p 8200:8200 \
  -v rlm-data:/data \
  ghcr.io/arman-kudaibergenov/rlm-workflow:latest
```

Проверить что работает:
```bash
curl http://localhost:8200/health
```

### Docker Compose

```bash
# Скачать конфиг
curl -O https://raw.githubusercontent.com/Arman-Kudaibergenov/rlm-workflow/main/docker/docker-compose.yml
curl -O https://raw.githubusercontent.com/Arman-Kudaibergenov/rlm-workflow/main/docker/.env.example
cp .env.example .env

# Запустить
docker compose up -d
```

По умолчанию используется FastEmbed (работает без GPU и без API-ключей).
Если хотите другой LLM-бэкенд — отредактируйте `.env` перед запуском.
Варианты описаны в [альтернативы-llm.md](альтернативы-llm.md).

## Вариант 2: pip (без Docker)

```bash
pip install rlm-toolkit
rlm-server --host 0.0.0.0 --port 8200 --data-dir ./data
```

## Вариант 3: удалённый сервер (для команд)

RLM лучше всего работает как постоянный сервер — его не нужно перезапускать между сессиями.

```bash
# На сервере (Ubuntu/Debian)
mkdir -p /opt/rlm
cd /opt/rlm
curl -O https://raw.githubusercontent.com/Arman-Kudaibergenov/rlm-workflow/main/docker/docker-compose.yml
curl -O https://raw.githubusercontent.com/Arman-Kudaibergenov/rlm-workflow/main/docker/.env.example
cp .env.example .env
docker compose up -d

# systemd автозапуск — если docker compose настроен с restart: unless-stopped,
# просто включите Docker в автозапуск:
systemctl enable docker
```

## Настройка Claude Code

Добавьте RLM в конфиг MCP (`~/.claude/mcp.json`):

```json
{
  "mcpServers": {
    "rlm-toolkit": {
      "type": "http",
      "url": "http://localhost:8200/mcp"
    }
  }
}
```

Для удалённого сервера замените `localhost` на IP сервера:
```json
"url": "http://192.168.1.100:8200/mcp"
```

Перезапустите Claude Code. Проверка:
```
claude mcp list
```
В списке должен появиться `rlm-toolkit`.

## Настройка хуков (автоматизация)

Хуки делают RLM-сохранение автоматическим — без ручных команд.

Подробная инструкция: [хуки.md](хуки.md)

Краткая версия:

```bash
# Скачать хуки в ~/.claude/hooks/
curl -o ~/.claude/hooks/pre-compact.ps1 \
  https://raw.githubusercontent.com/Arman-Kudaibergenov/rlm-workflow/main/examples/hooks/pre-compact.ps1
curl -o ~/.claude/hooks/context-monitor.ps1 \
  https://raw.githubusercontent.com/Arman-Kudaibergenov/rlm-workflow/main/examples/hooks/context-monitor.ps1
curl -o ~/.claude/hooks/auto-capture.ps1 \
  https://raw.githubusercontent.com/Arman-Kudaibergenov/rlm-workflow/main/examples/hooks/auto-capture.ps1
```

## Настройка CLAUDE.md

Скопируйте шаблон в корень проекта:

```bash
curl -o CLAUDE.md \
  https://raw.githubusercontent.com/Arman-Kudaibergenov/rlm-workflow/main/examples/CLAUDE.md.example
```

Замените секции `[PROJECT-SPECIFIC]` на описание вашего проекта.
Секции RLM-ритуалов (`контекст` / `суммаризируем` / `новая задача`) оставьте без изменений.
