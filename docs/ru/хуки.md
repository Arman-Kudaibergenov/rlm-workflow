# Хуки: автоматизация RLM

Хуки — это скрипты, которые Claude Code запускает автоматически при определённых событиях.
Три хука превращают ручные ритуалы в полностью автоматическое сохранение памяти.

## Как работает связка

```
Claude Code                    Хук                           RLM
────────────                   ────                          ───
/compact срабатывает  →  pre-compact.ps1  →  inject инструкция  →  Claude сохраняет в RLM

После каждого инструмента  →  context-monitor.ps1  →  проверить %  →  если ≥80% → суммаризируем

Edit/Write/Bash  →  auto-capture.ps1  →  запись в буфер  →  используется при суммаризируем
```

## Настройка в `~/.claude/settings.json`

```json
{
  "hooks": {
    "PreCompact": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "powershell -NoProfile -File \"$HOME\\.claude\\hooks\\pre-compact.ps1\""
          }
        ]
      }
    ],
    "PostToolUse": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "powershell -NoProfile -File \"$HOME\\.claude\\hooks\\context-monitor.ps1\""
          },
          {
            "type": "command",
            "command": "powershell -NoProfile -File \"$HOME\\.claude\\hooks\\auto-capture.ps1\""
          }
        ]
      }
    ]
  }
}
```

> **Linux / macOS:** замените `powershell -NoProfile -File` на `bash` и используйте `.sh`-версии скриптов.

## pre-compact.ps1 — сохранение перед compact

**Событие:** `PreCompact` — срабатывает перед `/compact` или авто-compact.

**Принцип:** хук выводит текст в stdout → Claude Code вставляет его как системное сообщение → Claude читает инструкцию и сохраняет в RLM.

Скрипт находится в [`examples/hooks/pre-compact.ps1`](../../examples/hooks/pre-compact.ps1).

Что он делает:
1. Генерирует timestamped инструкцию для Claude
2. Требует сохранить TASK PROGRESS и PENDING факты
3. Вызывает `rlm_sync_state()`

## context-monitor.ps1 — мониторинг контекста

**Событие:** `PostToolUse` — после каждого вызова инструмента.

**Принцип:** statusline.ps1 записывает текущий % контекста в `$TEMP\claude-ctx-state.json`. context-monitor.ps1 читает этот файл и при превышении порогов инжектирует инструкции.

**Пороги:**
- **70%** — предупреждение: "начинайте завершать задачу"
- **80%** — авто-суммаризируем: полный ритуал сохранения

Дедупликация: каждый порог срабатывает **один раз на сессию** (флаговые файлы в `$TEMP\claude-ctx-alerts\`).

При активной командной работе (TeamCreate вызывался в сессии) — сначала broadcast checkpoint всем агентам, ждёт сохранения, потом shutdown.

Скрипт: [`examples/hooks/context-monitor.ps1`](../../examples/hooks/context-monitor.ps1)

## auto-capture.ps1 — буфер изменений

**Событие:** `PostToolUse` — после каждого инструмента.

**Принцип:** молча записывает значимые операции в JSONL-буфер.

Что фиксирует:
| Инструмент | Что записывает |
|-----------|---------------|
| `Edit` | путь к изменённому файлу |
| `Write` | путь к новому файлу |
| `Bash` | команды git commit/push, сборки, тесты |

Формат записи:
```json
{"ts":"2026-02-28T10:30:15","tool":"Edit","file":"src/scheduler.py","session":"abc123"}
```

Хук никогда не блокирует — все ошибки подавляются (`SilentlyContinue`).

Буфер читается при `суммаризируем` и очищается после.

Скрипт: [`examples/hooks/auto-capture.ps1`](../../examples/hooks/auto-capture.ps1)

## statusline.ps1 — строка состояния

Не хук, но ключевая часть системы: показывает % использования контекста в строке статуса терминала.

```
claude-sonnet-4-6 | my-project | 85k / 200k | 42% used 85,432 | 58% remain 114,568 | thinking: Off
mcp: rlm-toolkit + 1c-help + playwright?
current: ●●●●●○○○○○ 45%  |  weekly: ●●●●●●●●○○ 82%
resets 3:45pm              |  resets feb 28, 11:59am
```

Без statusline context-monitor работает вслепую (читает из файла записанного statusline'ом).

Настройка для [Windows Terminal](https://learn.microsoft.com/windows/terminal/):
```json
{
  "profiles": {
    "defaults": {
      "tabTitle": "Claude Code"
    }
  }
}
```

И в профиле PowerShell (`.claude/statusline.ps1` запускается Claude Code автоматически если настроен в `settings.json`).

Скрипт: [`examples/hooks/statusline.ps1`](../../examples/hooks/statusline.ps1)

## Проверка работы

```bash
# Убедиться что хуки зарегистрированы
claude config list | grep hooks

# Проверить буфер захвата
cat ~/.claude/autocapture-buffer.jsonl

# Проверить файл состояния контекста
cat $TEMP/claude-ctx-state.json   # Windows
cat /tmp/claude-ctx-state.json    # Linux/Mac
```
